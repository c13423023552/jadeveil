# 设计笔记

几条贯穿整个主题的取舍，供想改造它的人参考。

## 分层

界面分两种材质：**玻璃**（左右侧栏、Ribbon、状态栏、菜单、命令面板等导航层）和**纸**（正文阅读面）。玻璃层用 `backdrop-filter` 真实透出桌面；纸面保持 ≥0.96 不透明——正文滚动路径上没有任何模糊层，这是性能和可读性的底线。窗口未开启透明、全屏、或打开"关闭全部玻璃"开关时，玻璃整体退化为同色实心面。

透明度是用户变量：不同壁纸的最优值不一样，所以浅色/深色各给了一个浓度滑杆（默认 0.64 / 0.85）。深浅公式刻意不对称——浅色玻璃靠低 alpha 就清透；深色 tint 在暗桌面上低 alpha 会塌成黑框，改为高 alpha + blur 透光影。

## 颜色

强调色是低饱和的青玉绿（色相 166°、饱和度 46%）。饱和度压低是为了让色相滑杆扫到任何位置都不出荧光色。checkbox、标签、选区、焦点环、callout 都从这一个色相派生。

浅色纸面是暖象牙色而不是纯白，整条灰阶跟着暖调；深色是带一点绿底温的石墨灰，侧栏和纸面落在同一明度带里，靠色相和玻璃光影分层而不是靠明暗差。

callout 九个语义色都压在低饱和带内，但色相间隔拉开了 15–25°，保证 note / info / tip / success 扫一眼能分清。

组件的"可见性"用海拔做（描边、投影、浮起），不引入新的底色——底色种类一多，玻璃和纸的两层结构就散了。

## 排版

面向中英混排：`text-autospace` / `text-spacing-trim` 按 `@supports` 渐进启用（代码区显式关闭）；行宽 42em（约 42 个全角字）；代码块行高 1.7 兼顾 CJK 注释；代码里禁用连字，`=>` 就显示成 `=>`。

标题不无限缩小：H5/H6 与正文同字号，靠字重和墨色深浅分级——缩得比正文小的"标题"会被读成注释。

## 对比度

正文约 13:1（浅）/ 11:1（深）；链接、语法高亮、callout 题字、任务态图标按 WCAG AA 校过。文件里所有写着"实测"的注释值都来自真机 computed style，不是目测。

## 维护约定

`theme.css` 文件头有一份维护契约：`--jadeveil-*` 是公共变量（Style Settings 绑定，不改名），`--jv-*` 是内部变量（可自由重构）；变体（OLED / 纸感加深 / 关闭玻璃）只允许重映射变量、不允许碰组件规则；每个 `!important` 都在行旁注明它对抗的 app.css 规则。改动前建议先读那份契约。
